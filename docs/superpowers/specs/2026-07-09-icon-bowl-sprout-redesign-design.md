# M27 图标重设计：碗+萌芽设计文档

> **版本**：v0.33.0
> **日期**：2026-07-09
> **变更类型**：UI 视觉精修（图标重设计）
> **前置版本**：v0.32.0（M26 图标：描边碗+米粒+右上飘叶+径向渐变）

## 1. 背景与目标

### 1.1 用户反馈

用户反馈当前 M26 图标两个问题：
1. **"图标好像是颠倒的"**：实际渲染碗口朝下（倒扣的碗）
2. **"那片叶子放的地方太突兀了"**：叶子飘在右上角 (80,40)，与碗无视觉连接感

### 1.2 根因分析（cairosvg 实测验证）

M26 的碗 path `M39,51 A15,15 0 0 1 69,51 Z` 中 `sweep-flag=1` 在 y-down 坐标系中实际渲染**弧线在上方**（y=36~51 区间），碗口在 y=51 朝下，碗底在 y=36 朝上——**碗确实倒扣**。

M26 注释写的"sweep=1 = 下方半圆 = 碗底"是错误判断。正确应为 `sweep=0`（弧线在下方，碗口朝上）。

连锁错误：米粒圆心 y=55-60 落在倒扣碗的"下方背景区"，实际不在碗内。

### 1.3 设计目标

- **修正颠倒**：碗口朝上、碗底朝下（sweep-flag 1→0）
- **叶子归位**：从右上角飘叶改为从碗中央萌芽向上生长，消除突兀感
- **语义升级**：碗(食物) + 萌芽双叶(健康/成长) = 健康饮食的生命力意象
- **精致打磨**：茎+双叶层次 + 配色深浅区分 + 比例协调，反复打磨
- **保留品牌延续**：主色自然绿 `#2E7D32`，描边碗+渐变背景延续 M26
- **monochrome 兼容**：Android 13+ 主题图标染色后层次仍可识别

## 2. 设计方案

### 2.1 用户决策链

### 2.3 几何规范（108×108 画布，安全区中心 54,54 半径 33）

#### 碗（修正 sweep）
- **填充 path**：`M39,51 A15,15 0 0 0 69,51 Z`（sweep 1→**0**）
- **描边 path**：同上，`strokeWidth=2.5` `strokeLineCap=round` 无填充
- 碗口 y=51（水平线，左 39 → 右 69），碗底 y=66（弧线在下方）
- 碗中心 x=54，碗宽 30dp，碗高 15dp
- **修正效果**：碗口朝上、碗底朝下（正放碗）

#### 米粒（2 粒，碗底对称）
- 左米粒：圆心 (50,61)，r=1.2dp
- 右米粒：圆心 (58,61)，r=1.2dp
- 圆画法：`M50,61 m-1.2,0 a1.2,1.2 0 1,0 2.4,0 a1.2,1.2 0 1,0 -2.4,0`
- 从 M26 的 3 粒减到 2 粒，让位萌芽焦点
- y=61 在碗内底部（碗口 51、碗底 66 之间）

#### 茎（从碗内中央底部向上生长）
- 直线 path：`M54,61 L54,38`
- 描边 `strokeWidth=1.5` `strokeLineCap=round`，主绿 `#2E7D32`
- 起点 (54,61) 在两粒米之间（碗内底部中央）
- 终点 (54,38) 在碗口上方 13dp（茎高 23dp）
- 比碗描边细 1dp，层次分明

#### 顶叶（茎尖，右上 30°）
- 水滴形 path：`M54,38 C52,35 52,31 62,30 C60,34 57,37 54,38 Z`
- 叶柄 (54,38) = 茎顶
- 叶尖 (62,30) = 右上 30°
- 主绿 `#2E7D32` 实心填充
- 第一条贝塞尔 (52,35)(52,31) → 叶面左上凸
- 第二条贝塞尔 (60,34)(57,37) → 叶背右下回弧

#### 侧叶（茎中部，左上 30°）
- 水滴形 path：`M54,46 C52,43 52,39 46,38 C48,42 51,45 54,46 Z`
- 叶柄 (54,46) = 茎中部
- 叶尖 (46,38) = 左上 30°
- `#388E3C` Green 600（比顶叶浅一阶，层次区分）
- 与顶叶左右对称（顶叶右上、侧叶左上）

#### 背景（保留 M26）
- 径向渐变：中心 (54,54) 白 `#FFFFFF` → 边缘 `#F1F8E9` Light Green 50
- 渐变半径 54dp

### 2.4 配色层次（5 色 + 渐变）

| 元素 | 颜色 | 色值 | 层次重量 |
|------|------|------|----------|
| 碗描边 | Green 900 深绿 | `#1B5E20` | 最重（轮廓） |
| 碗填充 | Green 50 淡绿 | `#E8F5E9` | 轻（衬托） |
| 茎 + 顶叶 | Green 800 主绿 | `#2E7D32` | 重（焦点） |
| 侧叶 | Green 600 中绿 | `#388E3C` | 中（副焦点） |
| 米粒 | Amber 200 淡黄 | `#FFF59D` | 中（食物对比色） |
| 背景中心 | White 白 | `#FFFFFF` | — |
| 背景边缘 | Light Green 50 | `#F1F8E9` | — |

相比 M26 新增 1 色（侧叶 Green 600），层次从 4 色 → 5 色，双叶深浅区分更精致。

### 2.5 视觉轴线与重心

- **轴线**：碗底 (54,66) → 茎尖顶叶 (62,30)，整体偏右上对角线
- **重心**：碗在下（重量稳），萌芽向上（轻盈），符合视觉重力
- **对称**：茎居中 x=54，双叶左右对称（顶叶右上、侧叶左上），不完全对称（顶叶更高）避免死板
- **焦点**：茎尖顶叶是视觉最高点，碗是视觉最重区，上下呼应

### 3.2 渲染脚本重写说明

`scripts/render_icon_png.py` 当前仍是 M15 叉刀几何（代码与实际 PNG 不符，PNG 实际是 M26）。本次重写为用 `cairosvg` 从 `ic_launcher_foreground.xml` + `ic_launcher_background.xml` 直接渲染 PNG，避免脚本与 XML 脱节。

渲染流程：
1. 读 `ic_launcher_background.xml` → 转 SVG → cairosvg 渲染背景层
2. 读 `ic_launcher_foreground.xml` → 转 SVG → cairosvg 渲染前景层
3. 合成两层 → 5 密度 PNG（mdpi 48 / hdpi 72 / xhdpi 96 / xxhdpi 144 / xxxhdpi 192）
4. 圆形蒙版生成 `ic_launcher_round.png`

注意：Android vector XML 转 SVG 需处理：
- `@color/xxx` 引用 → 读 `values/colors.xml` 解析为实际色值
- `<aapt:attr>` 内联渐变 → 转 SVG `<defs><radialGradient>`
- pathData 语法兼容（基本一致）

## 5. 风险与回退

### 5.1 风险

- **Android vector → SVG 转换**：`<aapt:attr>` 渐变语法需手动转 SVG `<defs>`，转换错误会导致背景渐变丢失
  - 缓解：渲染后像素采样验证背景色
- **cairosvg 与 Android vector 渲染差异**：cairosvg 渲染的 PNG 与真机 Android 渲染可能有细微差异（抗锯齿/描边端点）
  - 缓解：PNG 仅作 fallback，真机走 adaptive icon XML 渲染，PNG 差异可接受
- **monochrome 染色**：新增侧叶色 `#388E3C` 在 Android 13+ 主题染色时会丢失层次
  - 缓解：monochrome 模式下双叶都染色为同一色，靠形状轮廓识别，可接受

### 5.2 回退

若渲染异常，回退到 M26 XML（git revert），PNG fallback 不影响 minSdk=31 设备（全部走 adaptive icon）。
